# HASA 모델 API를 Cursor에서 직접 실행하는 실습 교안

## 0. 이 교안의 목표

이 자료는 HASA의 공개 모델 카탈로그를 단순히 소개하는 자료가 아니라, 수강생이 자신의 PC에서 다음 결과를 직접 만드는 실습 교안이다.

1. Cursor에서 Python 프로젝트와 가상환경을 준비한다.
2. HASA API 키를 환경변수로 안전하게 연결한다.
3. 텍스트·비전·임베딩·리랭커·음성 전사 모델을 실제로 호출한다.
4. 검색 결과를 근거로 답하는 최소 RAG를 만든다.
5. 마지막에 로컬 웹 채팅 앱을 실행한다.

이 교안은 필수 실습과 확장 실습이 분리되어 있다. 기본 120분 운영안으로 진행하거나, 필요한 모듈만 골라 1회차 교육에 사용할 수 있다.

## 2. 120분 예시 운영안

| 시간 | 모듈 | 수강생 산출물 |
|---:|---|---|
| 0~15분 | 환경 준비·API 키 연결 | Cursor 프로젝트와 .env |
| 15~30분 | 모델 확인·첫 채팅 | 내 키로 허용된 모델 목록과 첫 응답 |
| 30~45분 | 프롬프트·스트리밍·모델 비교 | 같은 질문의 모델별 차이 |
| 45~60분 | 비전 모델 | 이미지 또는 문서 사진의 구조화 결과 |
| 60~85분 | 임베딩·리랭커 | 의미 검색과 재정렬 결과 |
| 85~105분 | 최소 RAG | 근거 문서를 표시한 답변 |
| 105~120분 | 로컬 웹 앱·정리 | 브라우저에서 작동하는 채팅 앱 |

Whisper 실습은 15분 확장 모듈로 분리한다. 음성 전사 권한 또는 음성 파일이 준비되지 않은 경우에는 코드 설명과 실행 명령까지만 다룬다.

## 3. 시작 전 준비물

수강생 PC에는 다음만 있으면 된다.

- Cursor
- Python 3.11 또는 3.12
- HASA 개발 API 키
- 인터넷 또는 HASA API에 연결 가능한 네트워크

API 키는 반드시 HASA 포털의 API Keys 화면에서 본인이 발급받은 키를 사용한다. 개발용 키는 연동·프로토타입용이고, 실제 서비스에는 운영 승인을 받은 운영용 키를 사용한다.

### 교육자 사전 점검

1. HASA 포털에서 개발키가 발급되는지 확인한다.
2. 수업용 PC 한 대에서 아래 명령까지 실제로 통과하는지 확인한다.

    python 00_env_check.py

3. 비전 실습용으로 개인정보·민감정보가 없는 이미지 한 장을 준비한다.
4. Whisper 확장 실습을 할 경우 100MB 이하의 짧은 wav 또는 mp3 파일을 준비한다.
5. 모델별 권한 차이로 403이 날 수 있으므로, 텍스트 모델 하나와 비전 모델 하나를 대체 모델까지 정해 둔다.

## 4. Cursor에서 프로젝트를 여는 순서

압축 파일 hasa-cursor-model-lab.zip을 원하는 폴더에 풀고, Cursor에서 그 폴더를 연다. Cursor의 새 터미널에서 아래 명령을 순서대로 실행한다.

### Windows PowerShell

    python -m venv .venv
    .venv\Scripts\Activate.ps1
    python -m pip install --upgrade pip
    pip install -r requirements.txt
    Copy-Item .env.example .env

PowerShell이 실행 정책 오류를 보이면 현재 창에서만 아래 명령을 먼저 실행한 뒤 다시 활성화한다.

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
    .venv\Scripts\Activate.ps1

### macOS 또는 Linux

    python3 -m venv .venv
    source .venv/bin/activate
    python -m pip install --upgrade pip
    pip install -r requirements.txt
    cp .env.example .env

Cursor에서 Python 인터프리터를 선택할 때는 프로젝트 내부의 .venv를 선택한다. 터미널 앞에 (.venv)가 표시되면 준비가 된 것이다.

## 5. API 키와 모델 설정

.env.example을 복사해 만든 .env를 열고, HASA_API_KEY에 본인의 키만 입력한다. 키는 절대로 채팅창·소스코드·화면 캡처·PPT에 넣지 않는다.

    HASA_BASE_URL=http://210.123.135.174:8080/v1
    HASA_API_KEY=sk-dev-본인에게_발급된_키

나머지 모델 ID는 우선 기본값을 두고, 연결 확인 뒤 변경한다.

### 연결 확인

    python 00_env_check.py

정상이라면 현재 키로 사용할 수 있는 모델 ID 목록이 출력된다. 출력 목록에 없는 모델을 .env에 입력하면 403 오류가 날 수 있다.

### HASA API의 기본 구조

| 기능 | 엔드포인트 | 이 프로젝트의 파일 |
|---|---|---|
| 모델 조회 | GET /v1/models | 00_env_check.py |
| 채팅·비전 | POST /v1/chat/completions | 01~04, 09 |
| 임베딩 | POST /v1/embeddings | 05, 07 |
| 리랭커 | POST /rerank | 06 |
| 음성 전사 | POST /v1/audio/transcriptions | 08 |

HASA는 OpenAI 호환 API이므로 Python OpenAI SDK에서 base_url만 HASA 주소로 바꾸면 된다. 이 프로젝트의 hasa_client.py가 바로 그 공통 설정을 담당한다.

## 6. 실습 1 — 첫 채팅 API

01_chat.py를 열고, 메시지가 시스템 지시문과 사용자 질문으로 구성된 것을 먼저 확인한다. 그다음 아래를 실행한다.

    python 01_chat.py "초거대AI클라우드팜 API 교육의 학습 목표를 3개로 정리해줘."

### 확인할 것

- 응답이 터미널에 출력되는가
- 응답이 한국어로 나오며 요청한 3개 항목을 지키는가
- TEXT_MODEL을 바꾸면 답변의 문체·속도·구성도 달라지는가

### 프롬프트를 개선하는 4요소

좋은 프롬프트는 대개 아래 네 가지를 갖는다.

| 요소 | 나쁜 예 | 개선 예 |
|---|---|---|
| 목적 | 안내문 써줘 | AI 교육 신청 안내문을 작성해줘 |
| 입력 | 대충 알아서 | 대상은 대학생, 시간은 2시간, 준비물은 노트북 |
| 출력 형식 | 잘 써줘 | 제목 1개와 4개 항목의 Markdown 목록 |
| 검증 기준 | 빠뜨리지 마 | 날짜·대상·준비물·문의처가 모두 포함됐는지 마지막에 점검 |

Cursor에서 프롬프트를 고치기 전에는 prompts/CURSOR_PROMPTS.md의 3번 문구를 붙여 넣어 개선안을 먼저 받는다. 모델에게 코드 전체를 갑자기 고치게 하기보다, 목적과 출력 형식을 먼저 명확히 하는 습관을 익히는 것이 핵심이다.

## 7. 실습 2 — 스트리밍과 모델 비교

### 스트리밍

    python 02_stream_chat.py

질문을 입력하면 완성된 답을 한 번에 받지 않고, 생성되는 글자가 이어서 표시된다. 웹 서비스에서 사용자가 기다리는 느낌을 줄이는 기본 방식이다.

### 같은 질문을 여러 모델로 비교

    python 03_compare_models.py

권한이 없는 모델은 오류를 표시하고 다음 모델로 계속 진행한다. 모두 비교할 필요는 없다. 현재 키에 허용된 모델 중 두 개만 골라 아래를 비교하게 한다.

- 정확성: 사실과 추정을 구분하는가
- 형식 준수: 5개 항목, 표, 요약 등 요청한 형식을 지키는가
- 설명 방식: 너무 길거나 모호하지 않은가
- 속도: 응답이 늦는 이유가 모델 크기·GPU 자원일 수 있음을 이해하는가

### Cursor에서 사용할 비교 요청

    현재 03_compare_models.py의 실행 결과를 기준으로,
    모델별 답변을 정확성·형식 준수·간결성·추정 표현의 4개 기준으로 표로 비교해줘.
    실제 출력에 없는 성능 수치는 추정하지 마.
    코드는 수정하지 마.

## 8. 실습 3 — 비전 모델로 이미지 읽기

개인정보가 없는 이미지 파일을 준비한 뒤 아래처럼 실행한다.

    python 04_vision.py --image .\sample.png

문서 사진이라면 다음처럼 질문을 구체화한다.

    python 04_vision.py --image .\notice.png --question "이미지의 제목, 날짜, 대상, 준비물을 표로 정리해줘. 읽기 어려운 글자는 추정이라고 표시해줘."

04_vision.py는 이미지를 외부 URL에 올리지 않고 Data URL 형태로 API 요청에 포함한다. 실습 자료로는 민감한 계약서, 인사 문서, 개인 사진을 사용하지 않는다.

### 토론 질문

1. 이미지에서 사실로 읽힌 내용과 모델의 추정은 어떻게 구분할까?
2. OCR 결과를 바로 업무 시스템에 넣으면 안 되는 이유는 무엇일까?
3. 72B 비전 모델과 경량 비전 모델 중 어느 상황에 무엇을 고를까?

## 9. 실습 4 — 임베딩: 문장을 숫자 벡터로 바꾸기

    python 05_embeddings.py

임베딩은 문장의 의미를 숫자 배열로 바꾸는 과정이다. 05_embeddings.py는 한 질문과 세 문장을 bge-m3로 벡터화하고, 코사인 유사도로 가장 가까운 문장을 찾는다.

수강생에게는 생성형 AI가 바로 답을 만들어내는 단계와, 그 전에 관련 문서를 찾는 단계를 분리해 설명한다.

    질문 → 임베딩 → 유사 문서 찾기 → 관련 문서만 생성 모델에 전달 → 답변

이 구조가 다음 RAG 실습의 핵심이다.

## 10. 실습 5 — 리랭커: 관련 문서 순서를 더 정확히 만들기

    python 06_rerank.py

임베딩은 빠르게 후보 문서를 찾는 데 적합하고, 리랭커는 질문과 후보 문서를 한 쌍씩 더 자세히 비교해 순서를 다시 매기는 데 적합하다.

06_rerank.py는 HASA의 /rerank 응답 원본을 그대로 출력한다. 수강생은 index와 score를 보며 어떤 문서가 위로 올라갔는지 확인한다. 실제 서비스에서는 임베딩으로 상위 10개를 찾고, 리랭커로 상위 3개를 골라 생성 모델에 전달하는 구조를 많이 사용한다.

## 11. 실습 6 — 최소 RAG 완성

data/knowledge.txt를 열어 네 개의 짧은 문서를 확인한다. 그다음 아래를 실행한다.

    python 07_rag.py "API 키는 어디에 보관해야 하나요?"

다른 질문도 해 본다.

    python 07_rag.py "403 오류가 나면 무엇을 먼저 확인해야 하나요?"

### 수강생이 확인할 결과

1. 먼저 검색된 근거 문서와 유사도 점수가 보인다.
2. 그 문서만 모델에 전달된다.
3. 최종 답변 문장 끝에 근거 문서 번호가 붙는다.

### 확장 과제

data/knowledge.txt에 기관의 공개 FAQ를 한 문서씩 추가하고 같은 질문을 다시 실행한다. 답변이 바뀌었는지, 근거 번호가 올바른지 사람이 확인한다.

RAG는 환각을 없애는 기능이 아니라, 모델이 참고할 근거의 범위를 좁히는 방식이다. 따라서 원문 품질·검색 품질·최종 검수는 계속 필요하다.

## 12. 확장 실습 — Whisper 음성 전사

Whisper 모델 권한과 짧은 음성 파일이 준비된 경우 실행한다.

    python 08_transcribe.py --audio .\sample.mp3 --language ko

현재 공개 안내 기준으로 기본 음성 전사는 파일당 최대 100MB이고, mono 16kHz wav 또는 mp3가 권장된다. 전사 결과는 회의록 초안처럼 활용하되, 고유명사·숫자·날짜를 원본 음성과 대조한다.

## 13. 실습 7 — 로컬 웹 채팅 앱

다음 명령을 실행한다.

    streamlit run 09_web_app.py

터미널에 표시되는 로컬 주소를 브라우저에서 열면 간단한 채팅 창이 보인다. 사이드바의 모델 ID에는 00_env_check.py에서 확인된 허용 모델을 입력한다.

이 실습의 목적은 완성도 높은 서비스 배포가 아니라, 다음 흐름을 직접 연결해 보는 것이다.

    브라우저 입력 → Python 코드 → HASA API → 모델 응답 → 브라우저 표시

### Cursor에 요청할 다음 기능

    현재 09_web_app.py에 대화 내용을 모두 지우는 버튼을 추가하고 싶다.
    먼저 변경 계획을 3줄 이내로 제시하고, 그 뒤 최소한의 코드만 수정해줘.
    .env와 API 키 처리 방식은 절대 변경하지 마.
    수정 뒤 실행 명령과 확인 방법을 알려줘.

## 14. 오류를 수업에서 다루는 법

| 코드·증상 | 가장 먼저 볼 것 | 조치 |
|---|---|---|
| 401 | .env의 HASA_API_KEY | 키 누락·오타·공백 확인 |
| 403 | 모델 권한 | 00_env_check.py 출력에 모델이 있는지 확인 |
| 404 | 모델 ID·Base URL | .env의 주소와 모델 ID 확인 |
| 429 | 요청 한도 | 잠시 기다린 뒤 재시도, Retry-After 확인 |
| 503 | GPU 백엔드 상태 | 잠시 후 재시도, 포털 상태 확인 |
| ModuleNotFoundError | 가상환경·패키지 | (.venv) 활성화 후 pip install -r requirements.txt |
| 이미지 파일을 찾지 못함 | 경로 | 파일명·확장자·현재 폴더 확인 |

오류가 발생했을 때 Cursor에게 바로 고쳐 달라고 하기보다, prompts/CURSOR_PROMPTS.md의 2번 프롬프트로 원인과 확인 명령을 먼저 요청하게 한다. 이 과정을 통해 수강생이 코드 생성 도구를 단순 자동완성기가 아니라 검증 가능한 협업 도구로 사용하게 한다.

## 15. 교육 마무리 체크리스트

수강생은 아래 항목을 모두 해 보면 실습을 완료한 것이다.

- [ ] .venv와 .env를 만들고 키를 코드 밖에 보관했다.
- [ ] 내 키에 허용된 HASA 모델 목록을 조회했다.
- [ ] 텍스트 모델에 질문하고 스트리밍 응답을 받았다.
- [ ] 같은 질문을 두 모델 이상에 보내 차이를 비교했다.
- [ ] 이미지 한 장을 비전 모델에 전달했다.
- [ ] bge-m3 임베딩으로 의미 유사도를 계산했다.
- [ ] 리랭커 원본 응답을 확인했다.
- [ ] 근거 문서 번호가 붙는 RAG 답변을 만들었다.
- [ ] 선택적으로 Whisper 전사 또는 Streamlit 웹 앱을 실행했다.
- [ ] API 키·민감정보를 Cursor 채팅과 코드에 넣지 않았다.
