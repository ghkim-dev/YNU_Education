[문서 1] API 키 관리
개발 중에는 개발키를 사용하고, 실제 서비스에는 운영 승인 후 발급된 운영키를 사용한다.
키는 소스코드에 직접 쓰지 않고 .env 같은 환경변수 파일 또는 비밀관리 도구에 보관한다.

[문서 2] 모델 선택
텍스트 요약과 질의응답에는 한국어 업무 모델 또는 범용 채팅 모델을 우선 검토한다.
이미지·표·문서 사진을 함께 이해해야 하면 비전 모델을, 문서 검색에는 임베딩과 리랭커를 함께 사용한다.

[문서 3] RAG 기본 흐름
RAG는 사용자의 질문과 관련 있는 문서를 먼저 찾고, 그 문서만 생성형 AI에 함께 전달해 답을 만들게 하는 방식이다.
답변에는 어떤 문서를 근거로 했는지 표시하고, 원문과 결과를 사람이 다시 검토한다.

[문서 4] 오류 대응
401은 API 키 누락 또는 오류, 403은 모델 사용 권한 부족, 404는 모델 ID 또는 주소 오류를 뜻한다.
429는 요청 한도 초과이므로 Retry-After를 확인해 잠시 기다린 뒤 재시도하고, 503은 GPU 백엔드 준비 상태일 수 있다.
