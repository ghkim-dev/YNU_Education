# Cursor에 그대로 붙여 넣는 프롬프트

## 1. 프로젝트 구조를 이해시키기

    현재 폴더는 HASA OpenAI 호환 API 실습 프로젝트다.
    먼저 파일을 수정하지 말고, 각 Python 파일의 역할과 실행 순서를 7줄 이내로 설명해줘.
    API 키나 .env 파일의 실제 값은 절대로 출력하거나 변경하지 마.

## 2. 오류를 안전하게 진단하기

    아래 실행 오류를 분석해줘.
    1) 원인을 우선순위 순서로 3개 이내 제시하고
    2) 각 원인의 확인 명령을 알려주고
    3) 내가 승인하기 전에는 파일을 수정하지 마.

    [오류 붙여넣기]

## 3. 프롬프트 품질 개선하기

    01_chat.py의 사용자 질문을 더 좋은 업무용 프롬프트로 바꾸고 싶다.
    내 질문의 목적, 입력 정보, 출력 형식, 검증 기준이 빠진 부분을 지적한 뒤
    개선 프롬프트를 1개 제안해줘. 코드 파일은 수정하지 마.

## 4. 기능을 한 단계만 추가하기

    현재 프로젝트의 07_rag.py를 바탕으로, 답변에 사용한 문서 번호를 더 명확히 표시하고 싶다.
    변경 계획을 먼저 짧게 제시한 뒤, 최소한의 코드만 수정해줘.
    HASA_API_KEY, .env, Base URL은 수정하거나 출력하지 마.
    수정 뒤에는 실행 명령과 확인할 결과를 알려줘.
